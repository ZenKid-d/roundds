# Premium-коды (Boosty)

Подписка оформляется на Boosty, а доступ в приложении открывается **подписанным
кодом** (Ed25519). Приватный ключ — только у тебя; в приложение зашит лишь
публичный ключ, поэтому код **нельзя подделать** и **не нужен сервер**.

## 1. Один раз — свои ключи

В репозитории уже зашит рабочий публичный ключ (для проверки/тестов). Для
продакшена выпусти **свою** пару, чтобы приватный ключ был только у тебя:

```bash
dart run tool/premium_keygen.dart
```

- Публичный ключ вставь в `lib/core/premium/premium_config.dart` →
  `kPremiumPublicKey`.
- Приватный **seed** сохрани в надёжном месте (менеджер паролей). **Не коммить.**

## 2. Настрой Boosty и цену

В `lib/core/premium/premium_config.dart`:

- `kBoostyUrl` — ссылка на твою страницу Boosty.
- `kPremiumPriceLabel` — текст цены на экране Premium.
- `kSubscriptionDays` — длительность подписки по коду (по умолчанию 30).

## 3. Выдача кода подписчику (привязка к устройству)

Код привязывается к устройству — работает только на нём, одним кодом нельзя
поделиться. Поток:

1. Подписчик оплачивает на Boosty и открывает экран **Premium** → копирует
   **ID этого устройства** и присылает тебе.
2. Ты генерируешь код под этот ID:

```bash
export ROUNDDS_PRIV=<приватный seed из шага 1>
dart run tool/gen_license.dart --days 30 --owner "Имя" --device <ID_устройства>
```

3. Отправляешь код `RD1.…` — подписчик вставляет его на экране Premium.
   Проверка офлайн, срок и привязка зашиты в самом коде.

**Перенос на другое устройство:** подписчик присылает новый ID (с нового
устройства) — выпускаешь новый код с `--device <новый_ID>`.

Без `--device` код работает на любом устройстве (например, для тестов или
раздачи «поддержки» без привязки).

## Как это работает

- Код: `RD1.<base64url(payload)>.<base64url(signature)>`,
  payload — JSON `{"v":1,"exp":<unix_sec>,"own":"...","dev":"..."}`.
- Приложение проверяет подпись публичным ключом, сравнивает `exp` с текущей
  датой и `dev` с ID этого устройства. Просроченный или чужой код Premium не даёт.
- ID устройства — случайные 16 байт, хранятся в secure storage; переживают
  обновление приложения, сбрасываются при переустановке.

## Ограничения (честно)

- Привязка `dev` не даёт использовать один код на чужом устройстве, но **без
  сервера код нельзя отозвать досрочно**: после переноса старый код формально
  ещё действует на старом устройстве до конца срока. Гарантированный
  «единственный активный девайс» и мгновенный отзыв — только с бэкендом.
