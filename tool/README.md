# Premium-коды (Boosty)

Подписка оформляется на Boosty, а доступ в приложении открывается **подписанным
кодом** (Ed25519). Приватный ключ — только у тебя; в приложение зашит лишь
публичный ключ, поэтому код **нельзя подделать** и **не нужен сервер**.

## 1. Один раз — свои ключи

В репозитории уже зашит рабочий публичный ключ (для проверки/тестов). Для
продакшена выпусти **свою** пару, чтобы приватный ключ был только у тебя:

```bash
dart run tool/premium_keygen.dart
```

- Публичный ключ вставь в `lib/core/premium/premium_config.dart` →
  `kPremiumPublicKey`.
- Приватный **seed** сохрани в надёжном месте (менеджер паролей). **Не коммить.**

## 2. Настрой Boosty и цену

В `lib/core/premium/premium_config.dart`:

- `kBoostyUrl` — ссылка на твою страницу Boosty.
- `kPremiumPriceLabel` — текст цены на экране Premium.
- `kSubscriptionDays` — длительность подписки по коду (по умолчанию 30).

## 3. Выдача кода подписчику

После оплаты на Boosty генерируешь код на 30 дней и отправляешь его человеку:

```bash
export ROUNDDS_PRIV=<приватный seed из шага 1>
dart run tool/gen_license.dart --days 30 --owner "Имя подписчика"
```

На выходе — код `RD1.…`. Подписчик вставляет его на экране **Premium** в
приложении. Проверка идёт офлайн, срок зашит в самом коде.

## Как это работает

- Код: `RD1.<base64url(payload)>.<base64url(signature)>`,
  payload — JSON `{"v":1,"exp":<unix_sec>,"own":"..."}`.
- Приложение проверяет подпись публичным ключом и сравнивает `exp` с текущей
  датой. Просроченный код → Premium выключается автоматически.

## Ограничения (честно)

- Без сервера код **нельзя отозвать** досрочно и **нельзя привязать к устройству** —
  человек может передать его другому. Поле `own` (имя) — мягкий антишеринг
  (показывается в приложении). Для «поддержки» этого достаточно; если нужна
  жёсткая привязка/отзыв — потребуется бэкенд.
