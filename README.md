# Roundds

Единый Android-плеер, который проигрывает музыку из **YouTube Music**,
**SoundCloud**, **Яндекс Музыки** и **VK Музыки** внутри собственного плеера.
Тёмный «премиум» интерфейс на AMOLED-чёрном, с динамическим акцентом из обложки
и вращающимся винилом.

> ⚠️ **Дисклеймер.** Воспроизведение использует неофициальные методы извлечения
> потоков (в духе NewPipe/ViMusic). Это нарушает правила использования (ToS) этих
> сервисов и может затрагивать авторские права; методы могут ломаться при
> обновлениях сервисов, а токен Яндекса несёт риск блокировки аккаунта. Проект
> **не предназначен для Google Play** и публикуется как личный/учебный. Вся
> ответственность за использование — на пользователе. Обход DRM (например, для
> Spotify) намеренно не реализован.

## Возможности
- Единая лента и **агрегированный поиск** по всем подключённым сервисам.
- Полноэкранный плеер с **вращающимся винилом** и **динамическим акцентом** из обложки.
- Фоновое воспроизведение, уведомление с контролами, реакция на наушники (`just_audio_background`).
- Очередь (drag-reorder), плейлисты, история — локально на устройстве.
- Оффлайн: скачивание треков и плейлистов (с прогрессом в %), проигрывание из файла.
- Скробблинг в **Last.fm** (по желанию, вход в Настройках).
- Боковое меню со статусами источников.

## Стек
Flutter · Riverpod · go_router · just_audio · palette_generator · dio ·
youtube_explode_dart · google_fonts.

## Источники
| Сервис | Как играет | Что нужно |
|---|---|---|
| YouTube Music | внутри приложения (`youtube_explode_dart`) | ничего |
| SoundCloud | внутри приложения (публичный web `client_id`) | автоопределение |
| Яндекс Музыка | внутри приложения (прямой mp3) | OAuth-токен аккаунта |
| VK Музыка | внутри приложения (Kate Mobile API) | access-токен аккаунта |

> Spotify намеренно не входит в проект — требует обхода DRM.

## Сборка и запуск
См. [SETUP.md](SETUP.md). Кратко:
```bash
flutter pub get
flutter run            # на подключённый Android-девайс
flutter build apk      # APK в build/app/outputs/flutter-apk/
```
Выпуск релизов автоматизирован (сборка + публикация APK в репозиторий
обновлений) — см. [RELEASING.md](RELEASING.md).

## Архитектура
`domain/` — модели и интерфейс `MusicSource`; `data/sources/` — коннекторы +
`aggregator`; `playback/` — единый плеер на `just_audio`; `features/` — экраны
(home, player, search, library, drawer, settings); `core/` — тема, виджеты,
провайдеры.
