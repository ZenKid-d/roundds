# Roundds — планы (roadmap)

Список будущих функций. Не реализовано — это очередь задач.

## Импорт плейлистов из сервисов 🆕
Импорт существующих плейлистов пользователя в локальную медиатеку Roundds.

- **YouTube Music / YouTube**
  - Ввод ссылки на плейлист (или ID) → тянем треки через `youtube_explode_dart`
    (`yt.playlists.get(id)` + `yt.playlists.getVideos(id)`).
  - Каждое видео → `Track(source: youtube)`, сохраняем как локальный `PlaylistX`.
  - Возможен импорт по URL без авторизации (публичные плейлисты).
- **Яндекс Музыка**
  - При наличии токена — список плейлистов пользователя
    (`/users/{uid}/playlists/list`) и треки плейлиста
    (`/users/{uid}/playlists/{kind}` → trackIds → `/tracks`).
  - Выбор плейлиста для импорта → сохраняем как локальный `PlaylistX`.
- **UI:** в «Медиатеке» кнопка «Импорт» → выбор сервиса → ввод ссылки (YouTube)
  или выбор из списка (Яндекс) → прогресс импорта → новый плейлист.
- **Заметки:** импортируем метаданные/ссылки; воспроизведение — как обычно, через
  резолв потока на лету. Большие плейлисты импортировать порциями.

## Готово ✅
- Фоновое воспроизведение через `audio_service` (уведомление, lock-screen).
- Кросс-источниковый fallback (недоступный трек играем из YouTube).
- SoundCloud по OAuth-токену подписки (полные Go+ треки).
- Реальные **shuffle / repeat** (off/all/one).
- **«Избранное»** (лайк трека) + вкладка в Медиатеке.
- **Текст песен** (синхронный/обычный, lrclib.net) — экран с автоскроллом.
- **Скачивание/оффлайн** — загрузка треков, проигрывание из файла, вкладка «Загрузки».
