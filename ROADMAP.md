# Roundds — планы (roadmap)

## Готово ✅
- Фоновое воспроизведение через `audio_service` (уведомление, lock-screen).
- Кросс-источниковый fallback (недоступный трек играем из YouTube).
- SoundCloud по OAuth-токену подписки (полные Go+ треки).
- Реальные **shuffle / repeat** (off/all/one).
- **«Избранное»** (лайк трека) + вкладка в Медиатеке.
- **Текст песен** (синхронный/обычный, lrclib.net) — экран с автоскроллом.
- **Скачивание/оффлайн** — загрузка треков, проигрывание из файла, вкладка «Загрузки».
- **Предзагрузка** следующего трека (быстрое переключение).
- **Эквалайзер** (по полосам) + **таймер сна** (15/30/45/60 мин) — в меню «•••» плеера.
- **Страница артиста** (тап по имени артиста → его треки).
- **Статистика прослушиваний** (топ треков/артистов) — в Настройках.
- **Импорт плейлистов**: YouTube по ссылке + Яндекс из аккаунта (по токену).
- **Настройки «Внешний вид»**: фон (AMOLED/тёмно-серый), акцент (динамический/пресеты/свой
  цвет-пипетка), вид плеера (винил/обложка), уровень анимаций, скругления углов, шрифт.
- **Рекомендации**: персональные ряды на главной («Похожее на…», «Ещё от…», «Для вас»)
  на базе истории/лайков/статистики + эндпоинты похожих (Яндекс/SoundCloud) и подбор
  по артисту. **Радио** — бесконечный микс, очередь докручивается похожими на лету.
- **Импорт лайков YouTube Music** через официальный OAuth (YouTube Data API).
- **Кроссфейд-лайт** (затухание/появление на границах) + **скорость воспроизведения**.
- **Умные плейлисты** («Часто слушаю», «Недавнее», «Только лайки») + **бэкап**
  (экспорт/импорт библиотеки в JSON).
- **Радио по артисту** (кнопка на странице артиста) и **по жанру** (чипы на главной).
- **«Новинки и чарты»** — отдельный экран с чартом по каждому сервису.
- **Караоке-режим** — полноэкранный синхронный текст поверх размытой обложки.
- **Аним-фон плеера** — живой градиент из акцента (гейтится уровнем анимаций).
- **Гэпплес** — бесшовное воспроизведение через `ConcatenatingAudioSource`
  с предзагрузкой следующего трека.
- **Скробблинг Last.fm** — «сейчас играет» + скроббл по правилам (½ трека
  или 4 мин, треки короче 30 сек не считаются).
- **Прогресс скачивания в %** — по каждому треку и по плейлисту (кольцо с
  числом + строка «артист · NN%»).
- **Кэш ленты и поиска** (TTL) — повторный вход на главную и одинаковые
  запросы не ходят в сеть заново; pull-to-refresh и смена источников сбрасывают.
- **CI-релизы** — сборка и публикация APK в репозиторий обновлений автоматом
  (см. [RELEASING.md](RELEASING.md)).
- **Авто-перерезолв протухшего потока** — если ссылка истекает посреди трека,
  плеер прозрачно резолвит свежую и продолжает с той же позиции (бюджет попыток
  на трек, оффлайн-файлы не трогаются).
- **Надёжные загрузки** — YouTube качается нативно через youtube_explode
  (чанками, обходит 403 на полном GET); HLS-only треки SoundCloud скачиваются
  через YouTube-эквивалент; результат проверяется на целостность (размер).
- **Обход троттлинга YouTube** — при `RequestLimitExceededException` (частый на
  общем VPN-IP) резолв автоматически уходит на зеркала Piped; после лимита
  10 минут резолвим сразу через Piped, не тратя попытку на прямой путь.
  Зеркала выбираются «умно»: держимся рабочего, недавно упавшие обходим
  (кулдаун 5 мин), опрашиваем группами по 3 параллельно (быстро, когда часть
  лежит). Кэш резолва потока (по videoId) снижает число запросов и сбрасывается
  при ошибке воспроизведения, чтобы перерезолв брал свежую ссылку.

## Бэклог
- Настоящий кроссфейд с наложением (нужны два плеера) — сейчас fade-лайт/гэпплес.
- Локальные файлы устройства (нужны права + скан медиатеки).
- Альбомные страницы (тап по альбому → треклист).
- Виджет на экран / плитка в шторке; Android Auto (нативное, нужен тест на устройстве).
- (Вне проекта) Spotify — невозможен без обхода DRM; VK — не выбран.

## Техдолг: KGP + мажорные зависимости
Сборка предупреждает: `package_info_plus` и `share_plus` применяют старый Kotlin
Gradle Plugin — «Future versions of Flutter will fail to build». Это **риск
будущего**, не текущая поломка (сборка 1.5.2 рабочая).

Быстрый фикс невозможен: KGP-совместимые версии этих плагинов требуют **win32 ^6**,
а `file_picker` (<12, стабильных нет — только beta) и `flutter_secure_storage 9`
(через `flutter_secure_storage_windows`) ещё на **win32 ^5**. `dependency_overrides:
win32` НЕ работает — Flutter компилирует в kernel snapshot весь Dart-граф, включая
Windows-реализации плагинов, и они не собираются против чужой версии win32
(проверено: релиз v1.5.3 упал на этом).

**Апгрейд применён** (согласованный набор на win32 ^6, Flutter 3.44.4 свеж) —
pub get + analyze + test + сборка debug-APK зелёные в CI, т.е. Android
компилируется:
- `package_info_plus ^10.2.0`, `share_plus ^13.2.0` (миграция кода на
  `SharePlus.instance.share(ShareParams(...))`) — цель KGP;
- `flutter_secure_storage ^10.3.1` (windows 4.2 → win32 ^6);
- `file_picker ^12.0.0-beta.7` — **бета** (стабильной с win32 ^6 пока нет),
  миграция на `FilePicker.pickFile()`.

⚠️ **Перед релизом — проверка на реальном устройстве** того, что CI не ловит:
- **secure_storage 10**: хранилище переписано; миграция автоматическая
  (`migrateOnAlgorithmChange`), но `resetOnError` теперь true → при сбое токены
  сотрутся (повторный ввод Яндекс/SoundCloud). Плюс требование **minSdk ≥ 23**.
- **file_picker beta**: проверить выбор JSON при импорте плейлиста/бэкапа.
- **share_plus 13**: проверить «поделиться карточкой» и экспорт бэкапа/плейлиста.
Заодно можно пройтись по остальным мажорам (`riverpod` 3, `go_router` 17) —
тоже ломающие API.

Профилактика: CI теперь собирает debug-APK на каждый push — этот класс поломок
(kernel snapshot / win32) ловится до релиза, а не в релизной сборке.

Отдельно: `palette_generator` (динамический акцент) объявлен **discontinued** —
рабочий, но без обновлений; замена изменит подбор цвета. Держать на радаре.
